## 트러블 슈팅

1. 기본적인 책을 미리 추가하여 조회 하는기능에서 문제가 있습니다.
일반적인 System.out.println(bookList); 을 사용하면 주소값이 보이는데 여기서 Book 클래스에서 get기능을 추가하여 문자열이 보이게 만드는것 가지는 알았으나,
System.out.println(bookList.getTiltle()); 이런방식으로는 배열에 접근이 안되었고 
 System.out.println(bookList.get(0).getTitle()); 중간에 배열의 몇번째 값인지 알려주는 것을 추가해주어야 출력이 가능했습니다.

2. 대여를 했는지 여부를 표시하고 바꿔주기 위해 Book 클래스에서 set 을 추가한다는거 까지는 알았는데 어떻게 Book 속성의 3번째 값을 접근해야 하는지 몰라 한참을 검색해본 결과 
이것 역시 bookList의 get(배열값) 으로 먼저 접근한 다음  if 문을 이용해 현재 대여 여부에 대한 글을 equals() 로 비교 후 set(대여 여부)을 사용하여 바꿔주는 방식으로 만들게 되었습니다.

3. 책 등록기능에서 어떻게 List에 제목과 문자를 추가해서 넣어야 하는지에 대한 고민도 많았습니다.
일단 입력값을 변수명에 넣는것 까지는 생각했으나 그걸 어떻게 배열에 집어넣을지 한참 생각하다  혹시 그냥 변수도 Stirng 이니 
Book book1 = new Book("수학책", "수학선생님", "대여 가능"); 괄호 안에 속성값에 대신하여 변수명을 집어넣으니 -> bookList.add(new Book(resisterBookTitle, resisterBookAuthor, "대여 가능")); 
작동을 잘 하였습니다. 대여 여부의 경우 그냥 기본값을 대여 가능으로 집어넣은 상태에서 만들어 지도록 하였습니다.

지금 과제를 하면서 List에서 값을 불러오고 수정하고 추가하는 기능들에 대해 많은 공부가 되었습니다.
또 속성,생성자,기능에 () 안에 들어가는 인자 값이 어떤 역할을 하는지 조금 더 자세히 알아볼 수 있었고  
기능을 최대한(제가할 수 있는선에선) 분리시켜 놓아 관리하기 좋고 재활용할 수 있도록 만드는 법을 다시 공부 하였습니다.
